Few Point from GPT

  • The constructor sets up the subscription, but the observable itself will continue emitting values over time.
  • Observables are asynchronous and can emit multiple values (or even no value), so the this.exclusive will update whenever a new value is emitted from the exclusive observable.
  • The subscription is active as long as the component exists, and Angular will automatically clean it up when the component is destroyed, assuming you properly manage unsubscriptions (more on that below).
  • Observables are not one-time values; they can emit multiple values over time.
  • So, while the constructor runs only once, the subscription continues to listen for updates whenever the exclusive observable emits a new value.

IMP

  • Observables are passive and don't emit until someone subscribes.
  • Subjects can actively emit values with .next(), and the emitted values are broadcasted to all current subscribers.
  • The observable is like a radio station that broadcasts signals but doesn't care if anyone is listening.
  • The subject is like a radio host who can not only broadcast but also receive calls from listeners and interact with them.
